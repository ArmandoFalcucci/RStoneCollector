RStone

A wizard-style data entry app for lithic analysis. One question at a time, conditional fields, your data in plain CSV.

1. Install

You need R (any recent version) and optionally RStudio. Git is recommended for sharing and versioning your data.

  1. Install R: posit.co/download
  2. Install Git (optional but recommended): git-scm.com
  3. Clone the repo and install R dependencies:
git clone https://github.com/YOU/RStone.git MyProject
cd MyProject
# in R/RStudio from this folder:
source("install.R")
shiny::runApp()
Tip: if you want to use RStone for a different project later, just clone again into a separate folder. Each clone is fully independent — its own data, schema, and config.

2. First run

The first time you launch the app, a welcome dialog appears. Fill in:

  • Site / project name — anything you like; shown in the header bar.
  • Analyst name — recorded in the audit log so you can track who made which changes.
  • Data file — the CSV where your records will live (default: data/data.csv).
  • Schema — the CFG file defining which questions to ask. default.cfg is a good starting point. [example] boomplaas.cfg shows a richer worked example with the Flake/Core split.

You can change any of these later in Settings, or hit Run Welcome Setup there to start fresh.

3. Daily use — data entry

The wizard asks one question at a time. Move through the fields by:

  • Pressing Enter on text/numeric fields (or click Next →).
  • Clicking an option button — the wizard advances immediately.
  • Picking from a dropdown — same, auto-advances.
  • Pressing Shift+Enter to go back one step (works multiple times).
  • Pressing Esc to skip non-required fields.
  • Using the arrow keys to move between option buttons in a grid layout.

Conditional fields

Fields that don't apply are silently skipped. If you select RawMaterial = Indeterminate, the wizard will skip fields that depend on knowing the material. Change the condition retroactively (via Back) and any now-invalid downstream values are cleared automatically — you don't need to wipe them by hand.

Current Record panel

The right-hand panel shows everything you've already confirmed for the current record, in real time. The field you're currently typing in is highlighted in yellow. Carry-forward values from the previous record (Square, Unit, etc.) pre-fill the inputs but stay hidden until you confirm them, so the panel always starts clean on a new record.

Automatic save

When you reach the end of the applicable fields, the record is saved automatically — no review step. A new record starts immediately, and if your ID ends in digits it auto-increments (BPA-001 → BPA-002).

Editing past records: Edit Last Record (sidebar) or Edit Selected in View Records opens a modal where you can change one field or all fields, with the same validation as new entries. Duplicate creates a fresh record pre-filled from a chosen one.

4. The schema (CFG)

The schema is a plain text file in schemas/. Each block defines one field:

[RawMaterial]
TYPE=MENU
PROMPT=Select the raw material:
MENU=Quartzite,Quartz,Chert,Silcrete,Other
REQUIRED=True
CARRY=True

The Schema tab has four sub-tabs:

Editor

Edit the CFG text directly. Validate before saving — invalid CFGs cannot overwrite your file.

Add Field

Form to append one field with prompt, type, options, ranges, and conditions.

Edit Field

Pick an existing field and change any of its properties through the form.

Reorder / Delete

Change field order or remove a field (existing data in the CSV is preserved).

Field types

TypeUse for
TEXTIDs, free-form short strings (with optional regex PATTERN=)
NUMERICMeasurements (with optional MIN= / MAX=)
MENUCategorical options (define with MENU=A,B,C)
BOOLEANYes/No shorthand
NOTEMulti-line observations
IMAGEPath to a photo (shows inline preview if file exists)

Conditions

A field can be shown only when other fields take certain values:

[Notes]
TYPE=NOTE
PROMPT=Notes:
CONDITION1=HasNotes Yes

Combine conditions: by default AND, add OR at the end of a line to OR-combine with the next.

CONDITION1=Class Tool OR
CONDITION2=Class Core
CONDITION3=Material NOT Indeterminate

Full syntax reference: see schemas/README.md in your project.

5. Reports

Three sub-tabs:

Overview

Auto-generated bar charts for menu fields, violin+box plot of all numeric variables, summary statistics table. Three-slot filters let you slice the data (e.g. one square at a time). Download a multi-page PDF with cover, plots, stats, and any captured custom plots, or export just the summary statistics table as CSV.

Build Your Own

Point-and-click report builder. The controls adapt to the plot type:

Plot typeWhat you supply
Bar / Stacked / Grouped / ProportionalX categorical (+ group for stack/grouped)
Boxplot / ViolinX categorical, Y numeric (optional group)
Density / HistogramNumeric variable (optional group)
ScatterX numeric, Y numeric (optional color)

Facet splits the plot into small multiples by any categorical variable. Quick presets pre-fill the controls for common analyses. Three ways to take results out of the app:

  • Add to PDF report queues the plot for the next PDF download (cover page + all captured plots in one document).
  • PNG / JPG 300 dpi exports the current plot as a high-resolution image, ready for publication.
  • Table CSV exports the underlying cross-tabulation or summary table.

PCA & Correspondence Analysis

Two dedicated tabs for multivariate analysis. PCA: pick numeric variables, optionally color by category; scores, scree, and loadings. CA: pick two categorical variables; biplot of row+column points and inertia per dimension. Each tab has its own PNG/JPG 300 dpi export, and PCA also exports the loadings as CSV. Either can be captured into the PDF report.

Filters are shared. Whatever you set in the three-slot filter on the Overview tab also applies to Build Your Own, PCA, and CA. A status banner at the top of each tab shows the active filter and the resulting record count, so you always know what data is being analyzed.

6. Data files & backups

Every save writes two files side-by-side:

  • data/MySite.csv — primary data, git-friendly, opens in Excel/R/anything.
  • data/MySite.json — parallel JSON dump, preserves structure.

Plus:

  • backups/ — rolling backups (last 10 timestamped copies, oldest auto-pruned).
  • audit.log — JSONL log of every save, update, delete, and mass-edit with timestamp and analyst.
  • log/rstone.log — debug log.

If something goes wrong, you can always restore the most recent backup. The atomic write pattern (write to .tmp, then rename) means a crash never corrupts the main file.

7. Sync & share

The whole project — code, config, data, audit — sits in one folder, so you can put it under git as-is.

cd MyProject
git init
git remote add origin https://github.com/YOU/MyProject.git
git add .
git commit -m "Initial setup"
git push -u origin main

Then from the Export & Git tab inside the app:

  • Pull from GitHub — fetches the latest data, reloads.
  • Commit & Push — stages CSV + JSON + audit.log, commits with a descriptive message, pushes.

Each user/lab keeps their own repo. Multi-machine work is just git pull on the other machine.

9. FAQ

The app says my schema has an error and won't save. What now?

Read the red toast — it tells you which field/line is wrong. Your CFG file on disk is unchanged. Fix the textarea content and click Apply & Save again, or click Reload from File to discard your edits.

I see "Locked by another process".

Another RStone instance has the same CSV open (or one crashed and left a stale lock). If you're sure no one else is editing, delete the .lock file next to your CSV.

Why is saving the last field slower than the others?

That's when the full save happens: validate everything, rotate backups, atomic-write CSV and JSON, append the audit log. Usually well under a second; on slow disks or large datasets a bit more. This is durability, not waste.

What happens to a field's data if I delete the field from the schema?

The column stays in the CSV (so historical data isn't lost) but the wizard stops asking about it. Removing the column from the CSV by hand is safe if you decide later you want to clean it up.

How do I move between projects?

Either clone the repo into a separate folder per project, or change Database file and Active schema in Settings and click Save Settings.

Does dark mode persist?

Yes — saved in config.json.

Can I edit the CFG file outside the app?

Yes — any text editor. The app reloads it the next time you launch or when you click Reload from File.
